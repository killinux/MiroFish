# MiroFish 使用教程

以"大学食堂涨价舆论预测"为例，演示完整操作流程。

## 准备工作

确保服务已启动：
- 前端：http://localhost:3000
- 后端：http://localhost:5001

`.env` 中已配置好 `LLM_API_KEY` 和 `ZEP_API_KEY`。

## 第一步：首页 — 上传材料 + 填写需求

浏览器打开 `http://localhost:3000`，页面右栏是操作区：

1. **上传文件**：点击虚线上传区域，选择种子材料文件（支持 PDF / Markdown / TXT），也可以直接拖拽进去。示例文件：`docs/example_seed.txt`
2. **填写模拟需求**：在下方文本框中用自然语言描述你的预测需求，例如：
   ```
   预测食堂涨价事件的舆论走向
   ```
3. 点击底部黑色按钮 **「启动引擎」**

系统自动跳转到流程页面。

## 第二步：图谱构建（Step 1）

跳转后系统自动开始处理，无需手动操作：

### 阶段 1：本体生成（约 10-30 秒）

- 系统调用 LLM 分析上传的文本
- 自动识别实体类型（人物、组织、事件等）和关系类型
- 完成后页面显示生成的实体/关系类型标签
- 可以点击标签查看详情（描述、属性、示例）

### 阶段 2：图谱构建（约 1-3 分钟）

- 文本分块写入 Zep 知识图谱
- 页面显示进度条（0% → 100%）
- 等待进度条到 100% 并显示绿色"完成"标记

右侧面板会实时渲染图谱的节点和关系可视化。

## 第三步：环境配置（Step 2）

图谱构建完成后自动进入环境配置，分 3 个小步骤：

### 2.1 模拟实例初始化

自动创建模拟实例，页面显示：
- Project ID
- Graph ID
- Simulation ID

### 2.2 生成 Agent 人设

- 系统从图谱中提取人物实体
- 为每个人物生成详细人设档案（人格、背景、行为倾向）
- 完成后显示 Agent 列表和统计：当前 Agent 数 / 预期总数 / 关联话题数
- 点击 Agent 卡片可查看完整人设

以食堂涨价为例，会生成以下 Agent：
| Agent | 身份 |
|-------|------|
| 李明 | 学生会主席，反对涨价 |
| 王建国 | 后勤处处长，解释涨价原因 |
| 张小花 | 普通学生，理解但希望控制幅度 |
| 赵大厨 | 校园美食博主，关注菜品质量 |

### 2.3 生成模拟参数

- LLM 自动规划时间线和事件注入计划
- 页面显示模拟配置，可以查看和调整
- **模拟轮数**建议设为 **5 轮**（省 token），正式使用可设 20-40 轮

## 第四步：模拟运行（Step 3）

点击开始后进入模拟运行页面，两个社交平台并行模拟：

### Info Plaza（Twitter 模拟）

顶部显示实时状态：
- **ROUND**: 当前轮数 / 总轮数
- **TIME**: 已用时间
- **ACTS**: Agent 行为总数

可用动作：POST / LIKE / REPOST / QUOTE / FOLLOW / IDLE

### Topic Community（Reddit 模拟）

同样显示实时状态，可用动作更多：
POST / COMMENT / LIKE / DISLIKE / SEARCH / FOLLOW / MUTE

### 观察模拟过程

中间区域实时展示每个 Agent 的行为流：
- 谁发了什么帖子
- 谁点赞/转发了谁
- 谁发表了什么评论

等待两个平台都显示完成勾号 ✓ 即可进入下一步。

## 第五步：报告生成（Step 4）

模拟完成后：

1. 点击 **生成报告** 按钮
2. Report Agent 自动工作：
   - 规划报告大纲
   - 调用 InsightForge（深度搜索）和 PanoramaSearch（广度搜索）工具检索图谱
   - 逐章节生成 Markdown 内容
3. 页面实时展示：
   - 已完成的章节内容（无需等全部完成即可阅读）
   - Agent 执行日志（每步推理、工具调用、查询结果）
   - 控制台日志
4. 报告完成后可点击 **下载** 获取完整 Markdown 文件

## 第六步：深度交互（Step 5）

报告生成完毕后解锁交互功能，可以与 Report Agent 对话：

**示例问题**：
- `学生会主席的态度最终有变化吗？`
- `哪些 Agent 最活跃？`
- `舆论是否出现了反转？`
- `后勤处的回应对舆论有什么影响？`
- `总结一下各方的核心诉求`

Agent 会自主调用检索工具查找图谱和模拟数据来回答。支持多轮对话，可以追问。

## 费用估算

以食堂涨价例子（4 个 Agent、5 轮模拟）为参考：

| 步骤 | LLM 调用次数 | 说明 |
|------|-------------|------|
| 本体生成 | 1 次 | 分析文本，生成本体定义 |
| 模拟配置 | 1-2 次 | 生成人设和参数 |
| 模拟运行 | ~20 次 | 4 Agent × 5 轮 |
| 报告生成 | ~10 次 | 大纲 + 分章节 + 工具调用 |
| **合计** | **~33 次** | |

使用 DeepSeek-Chat（约 ¥1/百万输入 token，¥2/百万输出 token），总费用 **不到 ¥1**。

增大规模（20 Agent × 40 轮 = 800 次调用）费用约 ¥10-20。

## 自定义场景

MiroFish 不限于舆论预测，你可以上传任意种子材料：

- **金融预测**：上传财经报告，需求写"预测该政策对股市的影响"
- **政策推演**：上传政策草案，需求写"模拟各利益方的反应"
- **小说推演**：上传小说章节，需求写"推演后续剧情走向"
- **产品反馈**：上传产品评测，需求写"预测用户对新版本的态度"

关键是种子材料中包含多个角色/立场，让 Agent 之间有互动空间。
